// CDU 1
Nombre: Registrar Usuario
Autores: Cristihan Albarran
Fecha: 06-18-2019
description: Permite registrar usuarios en la red social.
actores: Usuario, Sistema
Precondiciones: El usuario debe crear una cuenta para iniciar sesion
Flujo Normal: 
  [1]: el actor introduce nombre, usuario, email, contrasena y confirmar contrasena.
  [2]: el actor presiona el boton de registrarse.
  [3]: el sistema valida los campos y los guarda.
  [4]: el sistema redirecciona el actor al login.
Flujo Alternativo: 
  [1]: El sistema comprueba la validez de los datos, si los datos no son correctos o el usuario y email ya existen, se avisa al actor de ello para que los corrija.
Postcondiciones: el usuario ha sido registrado en la base de datos

// CDU 4

Autores: Cristihan Albarran
Fecha: 06-18-2019
description: Permite Editar un evento en la red social.
actores: Usuario, Sistema
Precondiciones: El usuario debe estar autenticado en el sistema y debe tener uno o mas eventos en la tab de borrador.
Flujo Normal: 
  [1]: el actor va al menu y seleciona la pagina de, mis eventos.

  [4]: el sistema emite una peticion para precargar la data inicial del evento.
  [5]: el actor elige un otra pagina en el submenu.
  [6]: el actor llena los campos correspondientes,
  [7]: el actor presiona el boton de guardar.
  [8]: el sistema valida los campos y redirecciona al actor a la pagina de mis eventos.

  [1]: El sistema comprueba la validez de los datos, si los datos no son correctos, se avisa al actor de ello para que los corrija.
Postcondiciones: Evento editado.

  [1]: El sistema comprueba el status del evento para mostrar el boton de publicar, si el status del evento, no es publicable, no se muestra el boton de publicalarl
Postcondiciones: Evento publicado.
